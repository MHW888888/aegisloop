# v0.3.4 - Friendlier First Run

This release focuses on onboarding and safer first-time use. It does not change the bridge dispatch model.

## Added

- A practical onboarding playbook: `docs/onboarding.md`.
- Extension panel **Start here** checklist.
- Extension panel **Use starter text** button.
- README and first-run links for users asking "what should I type first?"

## Clarified

- Use a dedicated runner thread for automation.
- Keep ordinary Q&A in Chat Mode or a separate ChatGPT thread.
- Start with **Arm one run** before trying **Arm loop**.
- Use briefing files for serious project branches.

## Why

AegisLoop should be understandable in 30 seconds and safe to try in a few minutes. This update makes the default path more explicit without hiding the advanced Run Capsule controls.
